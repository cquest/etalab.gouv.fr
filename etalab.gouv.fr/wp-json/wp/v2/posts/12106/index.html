Etalab lance aujourd’hui code.etalab.gouv.fr<\/a>, un site permettant de parcourir la liste des codes sources publi\u00e9s par des organismes publics.<\/p>\n\n\n\n

En mai 2018, la DINSIC a lanc\u00e9 la politique de contribution de l’\u00c9tat aux logiciels libres<\/a> qui \u00e9nonce deux principes forts :<\/p>\n\n\n\n

  1. Les agents publics <\/strong>qui le souhaitent peuvent librement<\/em> contribuer \u00e0 des logiciels libres existants, ils sont encourag\u00e9s \u00e0 le faire avec leur adresse professionnelle.<\/li>
  2. Les organismes publics <\/strong>doivent publier les codes sources des logiciels qu’ils d\u00e9veloppent ou font d\u00e9velopper, ces codes sources sont consid\u00e9r\u00e9s depuis la loi pour une R\u00e9publique num\u00e9rique comme des \u00ab documents administratifs \u00bb et sont soumis aux m\u00eames obligations que les bases de donn\u00e9es.<\/li><\/ol>\n\n\n\n

    Comment trouver les codes sources aujourd’hui ouverts par l’administration ?<\/h4>\n\n\n\n

    Dans la foul\u00e9e de cette publication, nous avons commenc\u00e9 \u00e0 recenser les comptes via lesquels les organismes publics ouvrent leurs codes, sur GitHub, GitLab, Framagit ou des forges d\u00e9di\u00e9es. Nous avons ainsi r\u00e9f\u00e9renc\u00e9 \u00e0 la main plus d’une centaine de comptes d’organisation, groupes ou forges publiques.<\/p>\n\n\n\n

    Au sein d’Etalab, Antoine Augusti a d\u00e9velopp\u00e9 un logiciel<\/a> qui rassemble les informations de ces d\u00e9p\u00f4ts de codes sources et nous avons publi\u00e9 cet inventaire des d\u00e9p\u00f4ts de code source sur data.gouv.fr<\/a>, permettant \u00e0 chacun d’analyser les donn\u00e9es. Antoine a aussi mis en place une API<\/a> qui donne acc\u00e8s \u00e0 ces informations et Bastien Guerry a d\u00e9velopp\u00e9 un site web pour permettre \u00e0 chacun de les parcourir facilement.<\/p>\n\n\n\n
