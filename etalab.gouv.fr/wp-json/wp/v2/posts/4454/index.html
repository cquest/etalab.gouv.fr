{"id":4454,"date":"2016-03-09T12:11:41","date_gmt":"2016-03-09T11:11:41","guid":{"rendered":"https:\/\/www.etalab.gouv.fr\/?p=4454"},"modified":"2016-03-09T14:23:27","modified_gmt":"2016-03-09T13:23:27","slug":"codeimpot-un-hackathon-autour-de-louverture-du-code-source-du-calculateur-impots","status":"publish","type":"post","link":"https:\/\/preprod.etalab.gouv.fr\/codeimpot-un-hackathon-autour-de-louverture-du-code-source-du-calculateur-impots","title":{"rendered":"[:fr]#CodeImpot : un hackathon autour de l’ouverture du code source du calculateur imp\u00f4ts [:]"},"content":{"rendered":"[:fr]\n

Pour la premi\u00e8re fois en France, une administration va au-del\u00e0 de l’ouverture des donn\u00e9es publiques et met \u00e0 disposition le code source d’un de ses calculateurs. La Direction g\u00e9n\u00e9rale des finances publiques<\/a> (DGFiP) ouvre ainsi \u00e0 tous le code source de son calculateur imp\u00f4t. Pour accompagner cette ouverture, Etalab et la DGFiP organisent conjointement les 1er et 2avril, un hackathon de deux jours : #CodeImpot<\/a>.<\/p>\n

<\/p>\n

L’ouverture du code source d’un calculateur par une administration, une premi\u00e8re en France<\/strong><\/p>\n

La Direction g\u00e9n\u00e9rale des finances publiques mettra \u00e0 partir du 1er avril le code source du calculateur imp\u00f4t \u00e0 la disposition de tous. Acte de transparence et de gouvernement ouvert<\/a>, cette ouverture permettra d’expliciter les modalit\u00e9s de calcul des imp\u00f4ts, dans l’esprit de l’article 2 du projet de loi num\u00e9rique<\/a>. Elle est aussi un gage d’innovation car de nouveaux usages pourraient en d\u00e9couler, \u00e0 l’image du micro-simulateur ouvert OpenFisca<\/a>, que ce code enrichira.<\/p>\n

Ce code source sera pr\u00e9sent\u00e9 dans sa version 2014 qui a permis de calculer le montant d’imposition de 37 millions de foyers fiscaux. Il s’attache uniquement \u00e0 la fiscalit\u00e9 des personnes. D\u00e9velopp\u00e9 par la DGFiP dans un langage d\u00e9di\u00e9, le langage M, ce code source est transform\u00e9\u00a0par Etalab en arbre syntaxique abstrait<\/a>\u00a0qui sert de base \u00e0 une compilation en langage Python. Ces deux d\u00e9clinaisons seront \u00e9galement mises \u00e0 disposition au d\u00e9but du hackathon.<\/p>\n

#CodeImpot, un hackathon de deux jours pour accompagner l’ouverture du code source <\/strong><\/p>\n

Pour accompagner l’ouverture et d\u00e9velopper des cas d\u2019usage, Etalab et la DGFiP organisent un hackathon autour de ce code source. Pourront en d\u00e9couler \u00e0 travers diff\u00e9rents ateliers des traductions de la \u00ab calculette imp\u00f4t \u00bb vers d’autres langages de programmation, permettant le\u00a0d\u00e9veloppement d’applications ou d’API Web, et des r\u00e9utilisations par des chercheurs et des \u00e9conomistes pour r\u00e9aliser des \u00e9valuations et des \u00e9tudes.<\/p>\n

Ce hackathon de deux jours aura lieu les 1er et 2 avril \u00e0 la fondation Mozilla<\/a> (16 Bis Boulevard Montmartre,<\/span> Paris<\/span> <\/a>75009<\/a>). D\u00e9veloppeurs, \u00e9conomistes, fiscalistes, venus de d’administrations, de startups ou encore de laboratoires : les participants int\u00e9ress\u00e9s sont invit\u00e9s \u00e0 s’inscrire aux deux jours du hackathon<\/strong><\/a>. Des agents de la DGFiP et les \u00e9quipes d’Etalab et d’OpenFisca seront pr\u00e9sents les accompagner dans leurs travaux. Les restitutions auront lieu le samedi 2 avril \u00e0 partir de 17h o\u00f9 un large public est \u00e9galement invit\u00e9 \u00e0 s’inscrire<\/a>.
\n<\/span><\/p>\n

Un Framapad<\/a> est ouvert<\/strong> \u00e0 la contribution de tous<\/strong> pour recenser en amont de l’\u00e9v\u00e9nement les id\u00e9es des participants. Ceux d\u00e9sirant travailler sur un sujet pr\u00e9cis durant le hackathon auront \u00e9galement l’opportunit\u00e9 de le pr\u00e9senter en deux minutes dans la matin\u00e9e du 1er avril afin de former un groupe de travail autour de celui-ci.<\/p>\n

En s’engageant dans le mouvement d’ouverture des codes sources des logiciels, la DGFiP ouvre la voie \u00e0 la transparence des calculs et algorithmes qu’ils produisent. Cette d\u00e9marche in\u00e9dite pourrait faire \u00e9cole dans d’autres domaines, comme le calcul des droits \u00e0 la retraite ou bien l’affectation des \u00e9l\u00e8ves dans l’enseignement sup\u00e9rieur.<\/p>\n

L\u00e9gende : la data-visualisation ci-dessus repr\u00e9sente les relations entre les variables du code source des imp\u00f4ts.
